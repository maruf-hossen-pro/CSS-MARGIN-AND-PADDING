01 — Foundation

The CSS Box Model

Every HTML element is a rectangular box. CSS wraps that box in four layers. Understanding these layers is the key to mastering layout and spacing.

🧠 Quick Rule to Remember

Margin = space outside the element — pushes other elements away.
Padding = space inside the element — pushes content away from the edges.

02 — Margin

What is Margin?

Margin creates transparent (invisible) space around an element's border. It pushes neighbouring elements away. The element's background colour does not fill the margin area — it stays transparent.

/* Apply margin to all four sides */
.box {
  margin: 20px;
}

/* Or control each side separately */
.box {
  margin-top:    10px;
  margin-right:  20px;
  margin-bottom: 10px;
  margin-left:   20px;
}

03 — Padding

What is Padding?

Padding creates space between an element's content and its border. Unlike margin, the padding area is filled by the element's background colour. More padding = more "breathing room" inside the box.

/* Apply padding to all four sides */
.box {
  padding: 20px;
}

/* Or control each side separately */
.box {
  padding-top:    10px;
  padding-right:  20px;
  padding-bottom: 10px;
  padding-left:   20px;
}

05 — Shorthand

The Shorthand Values

Instead of writing four lines, use shorthand. Values are read clockwise from the top: Top → Right → Bottom → Left. A helpful memory trick: TR ou BL e.

Number of Values Example What it means
1 value margin: 20px All four sides = 20px
2 values margin: 10px 20px Top & Bottom = 10px  |  Left & Right = 20px
3 values margin: 10px 20px 30px Top = 10px  |  Left & Right = 20px  |  Bottom = 30px
4 values margin: 10px 20px 30px 40px Top=10   Right=20   Bottom=30   Left=40 (clockwise) 
/* The same shorthand rule applies to padding */
.card {
  padding: 8px 16px;    /* 8px top/bottom, 16px left/right */
  margin:  24px 0;     /* 24px top/bottom, 0 left/right  */
}
06 — Units

Which Units Can You Use?

Margin and padding accept many CSS units. Here are the most common ones you'll encounter as a beginner.

.box {
  margin:  20px;      /* px   → fixed pixels (most common) */
  padding: 1.5rem;   /* rem  → relative to root font-size (usually 16px) */
  margin:  2em;      /* em   → relative to element's own font-size */
  padding: 5%;       /* %    → percentage of the parent's width */
  margin:  auto;     /* auto → browser calculates (great for centering) */
  margin:  0;        /* 0    → no unit needed when value is zero */
}
💡 Which unit should I use?

Use px for precise, fixed spacing. Use rem for scalable spacing that respects user font settings. Use % in flexible, responsive layouts. For most beginners, px and rem are all you need.

07 — margin: auto

Centering with margin: auto

Setting margin: 0 auto on a block element that has a defined width will center it horizontally on the page. This is one of the most commonly used layout techniques in CSS!

I am centered! 🎯
.centered-box {
  width:  55%;     /* Must have a width for auto to work! */
  margin: 0 auto;  /* top/bottom = 0, left/right = auto  */
}
🔑 Remember

margin: auto only works for horizontal centering on block-level elements. The element must have a defined width — otherwise it takes up the full width and there is nothing to center.

08 — Gotcha!

Margin Collapse

When two vertical margins (top and bottom) touch, they collapse into one — only the larger of the two is used. This surprises almost every beginner!

.box-a { margin-bottom: 40px; }
.box-b { margin-top:    20px; }

/* You might expect: 40 + 20 = 60px gap
   But the actual gap is: 40px (only the larger one counts!) */
⚠ Watch Out

Margin collapse only happens with top and bottom margins on block-level elements. Horizontal (left/right) margins never collapse. Padding never collapses either.

09 — Advanced

Negative Margins

Margins can be negative — this pulls elements toward each other, or even outside their parent container. Padding, on the other hand, cannot be negative.

Box A
Box B — overlapping!
.box-b {
  margin-left: -20px;  /* pulls Box B 20px to the left */
}

/* Padding cannot be negative — browsers ignore it: */
.box {
  padding: -10px;  /* ❌ Invalid — nothing happens */
}
10 — Quick Reference

Cheatsheet

Save or bookmark this section — it has everything in one place.

MARGIN

  • margin: 20px; all sides
  • margin-top: 10px; top only
  • margin-right: 10px; right only
  • margin-bottom: 10px; bottom only
  • margin-left: 10px; left only
  • margin: 0 auto; center block
  • margin: 10px 20px; v | h
  • margin: -10px; negative ✅ ok!

PADDING

  • padding: 20px; all sides
  • padding-top: 10px; top only
  • padding-right: 10px; right only
  • padding-bottom: 10px; bottom only
  • padding-left: 10px; left only
  • padding: 10px 20px; v | h
  • padding: 0; remove all
  • padding: -10px; negative ❌ invalid
📋 Key Differences at a Glance

Margin — outside the border · transparent · can be negative · can collapse vertically · use auto for centering
Padding — inside the border · fills with background colour · always positive · never collapses
